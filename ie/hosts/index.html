为什么修改 hosts 文件可以上 Google 等网站？

因为天朝防火墙是黑名单机制，也就是网站域名或服务器 IP 在黑名单内才会被屏蔽，而 Google 等大型网站由于有服务器集群，服务器 IP 无数，总有漏网之鱼。

只要找到可用 IP 加上使用 HTTPS 访问即可访问 Google 等网站。

hosts 文件是什么？

通常我们通过域名访问网站时是由 DNS 获取网站服务器 IP 地址，而 hosts 文件则允许我们指定访问的服务器 IP 地址，这样我们就可以给访问的域名指定一个服务器 IP 地址来实现访问指定服务器。

通过修改 hosts 文件来上 Google 等网站安全吗？

修改 hosts 来上 Google 或 Facebook 等网站时，请务必使用 https 加密链接访问，且不能忽略错误证书警告。加密链接的证书验证可以确保我们访问到的是真实可靠的网站，而不是被中间人监听或篡改的内容。

不安全的链接例子：http://www.google.com.hk

安全的加密链接例子：https://www.google.com.hk

hosts 文件在哪里？

以下列出常用操作系统的 hosts 文件位置。

Microsoft Windows NT、2000、XP、7、8、10

C:\Windows\System32\drivers\etc\hosts

如果系统盘或系统目录不一样，请注意修改。

Unix、类 Unix 系统（如 Linux、Ubuntu、Mac OS X 等）

/etc/hosts

如何打开和编辑 hosts 文件？

由于 hosts 文件名就是 hosts 而没有后缀，所以默认是没有打开方式的。我们只要选中点击右键选择“打开方式”再选择“记事本”就可以编辑了。（Mac OS X 选择的是“文本编辑”）

常用快捷键：Ctrl+A 是全选，Ctrl+C 是复制，Ctrl+V 是粘贴，Ctrl+X 是剪切，Ctrl+Z 是取消上一步操作。选择多行快捷键：点击一行，按住 Shift 键再点击另一行，即可选中包括这两行中间的每一行。

权限问题：如果直接编辑提示权限不足无法保存，可以先把文件复制到桌面编辑，编辑好了再替换文件。

如何修改 hosts 文件上 Google 等网站？

将可用 hosts 文件的内容复制粘贴到本机 hosts 文件内或直接替换即可。如果修改 hosts 后没有立即生效，清理一下 DNS 缓存或重启网络即可（一般是断开网络后重新连接）。

Microsoft Windows 清理 DNS 缓存

按下 Windows+R 快捷键后输入cmd打开命令提示符，输入ipconfig /flushdns回车。

Mac OS X 清理 DNS 缓存

打开终端，输入sudo killall -HUP mDNSResponder回车。

Linux 清理 DNS 缓存

打开终端，输入sudo rcnscd restart回车。

systemd 发行版输入sudo systemctl restart NetworkManager回车。

为什么不应该在移动设备上通过修改 hosts 上网？

由于移动设备的应用对 https 加密链接验证方式支持不友好，导致修改 hosts 后也只能访问部分内容，即使在修改 hosts 后使用 VPN 仍然会导致部分资源不可访问，出现大量无法加载的现象等。

而且移动设备普遍需要开放全部 root 权限才可以修改 hosts 文件，存在一定风险。所以不建议在移动设备上使用修改 hosts 文件的方式来访问 Google 等网站。

正确认识天朝防火墙以及 VPN 虚拟专用网

当前全球互联网是中美两家的天下，早年美国互联网攻城略地时中国互联网能坚守阵地且发展出自己的一片天，天朝防火墙发挥了不可或缺的作用。凡事有利也有弊，天朝防火墙在发挥积极作用时也带来了不少消极作用，如导致天朝大陆人无法正常访问 Google 等网站。总体来讲，尽管给大家带来了不便，但天朝防火墙对中国的贡献多于弊端。希望大家能理性看待防火墙的存在，不要盲目攻击天朝的防火墙。

使用 VPN 不是违法行为。清查 VPN 时查封及处理的是非法经营者，与普通用户没有关系。提供 VPN 也不是违法行为，违法的是那些没有进行相关注册和备案的非法经营者。
